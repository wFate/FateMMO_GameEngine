# SDF Text Rendering Implementation Plan

> **For agentic workers:** REQUIRED: Use superpowers:subagent-driven-development (if subagents available) or superpowers:executing-plans to implement this plan. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace bitmap font rendering with MTSDF text that scales perfectly, supports outline/glow/shadow effects, and renders through the existing SpriteBatch without extra draw calls.

**Architecture:** Pre-generated MTSDF atlas (msdf-atlas-gen) loaded at startup. Uber-shader with `renderType` vertex attribute branches between sprite path (0.0) and MSDF text paths (1.0-4.0 for different styles). SDFText class submits text quads to SpriteBatch using the font atlas as a regular texture. Old TextRenderer removed.

**Tech Stack:** C++23, OpenGL 3.3, GLSL, msdf-atlas-gen (offline tool), nlohmann/json, stb_image

**Spec:** `Docs/superpowers/specs/2026-03-17-sdf-text-design.md`

**Build:** `"/c/Program Files/Microsoft Visual Studio/18/Community/Common7/IDE/CommonExtensions/Microsoft/CMake/CMake/bin/cmake.exe" --build out/build --config Debug 2>&1 | tail -50`
**Tests:** `out/build/Debug/fate_tests.exe`
**CRITICAL: NEVER create .bat files. They hang indefinitely.**

## Task 1: Generate MTSDF Atlas Assets

**Files:**
- Create: `assets/fonts/default.png`
- Create: `assets/fonts/default.json`

- [ ] **Step 1: Install msdf-atlas-gen**

Download from https://github.com/Chlumsky/msdf-atlas-gen/releases. Extract to a local tools directory. This is a one-time offline tool, not a build dependency.

- [ ] **Step 2: Choose a font**

Use a clean, readable font suitable for a 2D MMO. Options: `Roboto-Regular.ttf`, `Inter-Regular.ttf`, or the system `consola.ttf`. Place the .ttf in `assets/fonts/`.

- [ ] **Step 3: Generate the atlas**

```bash
msdf-atlas-gen -font assets/fonts/YourFont.ttf -type mtsdf -format png \
    -imageout assets/fonts/default.png -json assets/fonts/default.json \
    -size 48 -pxrange 4
```

This generates a 512x512 (or auto-sized) MTSDF atlas covering ASCII + extended Latin.

If msdf-atlas-gen is not available or cannot be installed, create a **placeholder**: generate a simple bitmap atlas programmatically (or use the existing stb_truetype atlas as a temporary fallback) and a hand-written JSON metrics file for ASCII 32-126. The SDF shader can be tested with approximate data — the visual quality will be poor but the pipeline works.

- [ ] **Step 4: Verify assets exist and commit**

```bash
git add assets/fonts/default.png assets/fonts/default.json
git commit -m "assets: add MTSDF font atlas and glyph metrics"
```

## Task 2: SpriteBatch Uber-Shader Upgrade

**Files:**
- Modify: `engine/render/sprite_batch.h`
- Modify: `engine/render/sprite_batch.cpp`

- [ ] **Step 1: Read sprite_batch.h and sprite_batch.cpp fully**

Understand: SpriteVertex struct, how vertices are pushed, the embedded shader source strings, flush() GL calls, VAO/VBO setup in init().

- [ ] **Step 2: Add renderType to SpriteVertex**

In `sprite_batch.h`, add to `SpriteVertex`:
```cpp
struct SpriteVertex {
    float x, y;       // position
    float u, v;       // texcoord
    float r, g, b, a; // color tint
    float renderType;  // 0.0=sprite, 1.0=normal text, 2.0=outlined, 3.0=glow, 4.0=shadow
};
```

Add renderType parameter to `drawTexturedQuad`:
```cpp
void drawTexturedQuad(unsigned int glTexId, const SpriteDrawParams& params, float renderType = 0.0f);
```

Add renderType to `BatchEntry`:
```cpp
struct BatchEntry {
    std::shared_ptr<Texture> texture;
    unsigned int rawTexId = 0;
    SpriteDrawParams params;
    float renderType = 0.0f;
};
```

- [ ] **Step 3: Update ALL vertex construction in sprite_batch.cpp**

Search for every `vertices_.push_back({...})` or vertex construction. Add `, 0.0f` for renderType to EVERY existing sprite vertex. For text quads (from drawTexturedQuad with renderType != 0), use the passed renderType value.

This is the most error-prone step — missing even one push_back causes undefined behavior.

- [ ] **Step 4: Update VAO attribute setup in init()**

Find where `glVertexAttribPointer` is called. Add the new attribute for renderType:

```cpp
// Existing attributes (0=position, 1=texcoord, 2=color)...

// Attribute 3: renderType
glEnableVertexAttribArray(3);
glVertexAttribPointer(3, 1, GL_FLOAT, GL_FALSE, sizeof(SpriteVertex),
    (void*)(8 * sizeof(float)));  // offset = 8 floats (x,y,u,v,r,g,b,a)
```

Update stride to `sizeof(SpriteVertex)` if not already using it.

- [ ] **Step 5: Update the vertex shader**

Add `in float aRenderType;` input and `out float v_renderType;` varying. Pass through to fragment shader.

```glsl
layout(location = 0) in vec2 aPosition;
layout(location = 1) in vec2 aTexCoord;
layout(location = 2) in vec4 aColor;
layout(location = 3) in float aRenderType;

out vec2 v_uv;
out vec4 v_color;
out float v_renderType;

uniform mat4 uViewProjection;

void main() {
    gl_Position = uViewProjection * vec4(aPosition, 0.0, 1.0);
    v_uv = aTexCoord;
    v_color = aColor;
    v_renderType = aRenderType;
}
```

- [ ] **Step 6: Update the fragment shader with MSDF branch**

```glsl
in vec2 v_uv;
in vec4 v_color;
in float v_renderType;

uniform sampler2D uTexture;
uniform float u_pxRange;      // 4.0 (from atlas generation)
uniform vec2 u_atlasSize;     // vec2(512.0, 512.0)
uniform vec2 u_shadowOffset;  // vec2(1.0/512.0, -1.0/512.0)

out vec4 fragColor;

float median(float r, float g, float b) {
    return max(min(r, g), min(max(r, g), b));
}

void main() {
    if (v_renderType < 0.5) {
        // Sprite path
        fragColor = texture(uTexture, v_uv) * v_color;
    } else {
        // MSDF text path
        vec4 sdf = texture(uTexture, v_uv);
        float sd = median(sdf.r, sdf.g, sdf.b);

        vec2 unitRange = vec2(u_pxRange) / u_atlasSize;
        vec2 screenTexSize = vec2(1.0) / fwidth(v_uv);
        float screenPxRange = max(0.5 * dot(unitRange, screenTexSize), 1.0);
        float screenPxDist = screenPxRange * (sd - 0.5);
        float opacity = clamp(screenPxDist + 0.5, 0.0, 1.0);

        if (v_renderType < 1.5) {
            // Normal text
            fragColor = vec4(v_color.rgb, v_color.a * opacity);
        } else if (v_renderType < 2.5) {
            // Outlined
            float outlineDist = screenPxRange * (sd - 0.35);
            float outlineOp = clamp(outlineDist + 0.5, 0.0, 1.0);
            vec4 outline = vec4(0.0, 0.0, 0.0, v_color.a * outlineOp);
            vec4 fill = vec4(v_color.rgb, v_color.a * opacity);
            fragColor = mix(outline, fill, opacity);
        } else if (v_renderType < 3.5) {
            // Glow
            float glowOp = smoothstep(0.0, 0.5, sdf.a);
            vec4 glow = vec4(v_color.rgb, v_color.a * glowOp * 0.6);
            vec4 fill = vec4(v_color.rgb, v_color.a * opacity);
            fragColor = mix(glow, fill, opacity);
        } else {
            // Shadow
            vec2 shadowUV = v_uv - u_shadowOffset;
            float shadowSd = median(texture(uTexture, shadowUV).rgb);
            float shadowDist = screenPxRange * (shadowSd - 0.5);
            float shadowOp = clamp(shadowDist + 0.5, 0.0, 1.0) * 0.5;
            vec4 shadow = vec4(0.0, 0.0, 0.0, v_color.a * shadowOp);
            vec4 fill = vec4(v_color.rgb, v_color.a * opacity);
            fragColor = mix(shadow, fill, opacity);
        }
    }
}
```

- [ ] **Step 7: Set MSDF uniforms in flush()**

After binding the shader, set the MSDF-specific uniforms:
```cpp
shader_.setFloat("u_pxRange", 4.0f);
shader_.setVec2("u_atlasSize", 512.0f, 512.0f);
shader_.setVec2("u_shadowOffset", 1.0f/512.0f, -1.0f/512.0f);
```

If `setVec2` doesn't exist on Shader, add it or use `setFloat` twice.

- [ ] **Step 8: Build and verify sprites still render correctly**

The existing sprite rendering must work identically — all sprite vertices have `renderType = 0.0f`, so the shader takes the sprite path.

## Task 3: SDFText Class — Atlas Loading & Text Drawing

**Files:**
- Create: `engine/render/sdf_text.h`
- Create: `engine/render/sdf_text.cpp`
- Create: `tests/test_sdf_text.cpp`

- [ ] **Step 1: Create `engine/render/sdf_text.h`**

```cpp
#pragma once
#include "engine/core/types.h"
#include "engine/render/sprite_batch.h"
#include <string>
#include <unordered_map>
#include <cstdint>

namespace fate {

enum class TextStyle : uint8_t {
    Normal   = 1, // renderType 1.0
    Outlined = 2, // renderType 2.0
    Glow     = 3, // renderType 3.0
    Shadow   = 4  // renderType 4.0
};

struct GlyphMetrics {
    float advance;
    float bearingX, bearingY;
    float width, height;
    float uvX, uvY, uvW, uvH; // atlas UV rect (0-1 normalized)
};

class SDFText {
public:
    static SDFText& instance() {
        static SDFText s_instance;
        return s_instance;
    }

    bool init(const std::string& atlasPath, const std::string& metricsPath);
    void shutdown();

    // Draw in world space (Y-up, for damage numbers and nameplates)
    void drawWorld(SpriteBatch& batch, const std::string& text, Vec2 position,
                   float fontSize, Color color = Color::white(), float depth = 50.0f,
                   TextStyle style = TextStyle::Normal);

    // Draw in screen space (Y-down, for UI text)
    void drawScreen(SpriteBatch& batch, const std::string& text, Vec2 position,
                    float fontSize, Color color = Color::white(), float depth = 50.0f,
                    TextStyle style = TextStyle::Normal);

    // Measure text dimensions
    Vec2 measure(const std::string& text, float fontSize) const;

    unsigned int atlasTextureId() const { return atlasTexId_; }
    float pxRange() const { return pxRange_; }
    float atlasWidth() const { return atlasWidth_; }
    float atlasHeight() const { return atlasHeight_; }

    // Screen projection utility (carried over from TextRenderer)
    static Mat4 screenProjection(int windowWidth, int windowHeight);

private:
    SDFText() = default;
    unsigned int atlasTexId_ = 0;
    float atlasWidth_ = 512.0f;
    float atlasHeight_ = 512.0f;
    float pxRange_ = 4.0f;
    float lineHeight_ = 1.0f; // in em units
    std::unordered_map<uint32_t, GlyphMetrics> glyphs_; // keyed by Unicode codepoint

    void loadMetrics(const std::string& jsonPath);

    // Internal draw implementation (yFlip=false for Y-up, true for Y-down)
    void drawInternal(SpriteBatch& batch, const std::string& text, Vec2 position,
                      float fontSize, Color color, float depth, TextStyle style,
                      bool yDown);

    // UTF-8 decode helper — returns next codepoint, advances index
    static uint32_t decodeUTF8(const std::string& text, size_t& index);
};

} // namespace fate
```

- [ ] **Step 2: Create `engine/render/sdf_text.cpp`**

Implement:
- `init()` — load atlas PNG via stb_image (4 channels, GL_RGBA8, GL_LINEAR, GL_CLAMP_TO_EDGE), parse metrics JSON
- `loadMetrics()` — read msdf-atlas-gen JSON format, populate glyphs_ map. The JSON format has `"glyphs"` array with `"unicode"`, `"advance"`, `"planeBounds"` (bearing/size), `"atlasBounds"` (UV rect).
- `drawInternal()` — for each character: decode UTF-8 to codepoint, look up GlyphMetrics, compute quad position from pen + bearing + fontSize, submit via `batch.drawTexturedQuad(atlasTexId_, params, (float)style)`, advance pen by `advance * fontSize`
- `drawWorld()` — calls drawInternal with yDown=false
- `drawScreen()` — calls drawInternal with yDown=true
- `measure()` — iterate characters, sum advances, track max height
- `decodeUTF8()` — simple state machine (~30 lines) handling 1-4 byte sequences
- `screenProjection()` — same as TextRenderer's implementation
- `shutdown()` — delete atlas texture

- [ ] **Step 3: Create `tests/test_sdf_text.cpp`**

Test cases (using a mock/minimal metrics dataset, not the real atlas):
1. "UTF-8 decode ASCII" — decode "Hello", verify 5 codepoints 72,101,108,108,111
2. "UTF-8 decode extended" — decode "café", verify codepoint 233 for é
3. "measure basic text" — register mock glyphs, measure "AB", verify width = sum of advances
4. "TextStyle enum values match renderType" — verify Normal=1, Outlined=2, Glow=3, Shadow=4

```bash
git add engine/render/sdf_text.h engine/render/sdf_text.cpp tests/test_sdf_text.cpp
```
